## 2. 전략 구성 — 2-북 체제

### 전략 A · LS-Alpha (마켓 뉴트럴) — [STRATEGY.md](STRATEGY.md)

- **알파 원천**: ① 동일 섹터 내 어닝 리비전 격차 ② 구조적 디스카운트 평균회귀(우선주 괴리 등) ③ 카탈리스트 이벤트
- **노출도**: Gross 목표 180~200% (현재 빌드업 Phase 1, 페어 5개 / 40%), Net 0±5% **베타 조정 기준**
- **정책 상한(사용자 지정)**: Gross ≤ 300%, |Net| ≤ 50% — 운용 경보선은 220% / ±10%
- **초기 5페어**: 삼성전자L/하이닉스S(크라우딩 언와인드), 삼전우L/삼전S(괴리 31.9%), TSM L/INTC S, WMT L/TGT S, LG엔솔L/SDI S
- **리스크 룰(기계 집행)**: 페어 −8% 스톱 → 20일 플랫 / 숏레그 +15% → 50% 커버 / 월중 −3% → Gross 반감 / 월중 −5% → 북 청산 심사 / 일간 −1.5% → 신규 동결 / **12개월 스톱 3회 → 페어 영구 폐기(로테이션 룰)**
- **팩터 중립**: 스타일 7팩터 ±0.20σ soft / ±0.30 hard
- VaR 한도 1d99 ≤ NAV 1.2%

### 전략 B · LS-D (디렉셔널 L/S) — [STRATEGY_DIRECTIONAL.md](STRATEGY_DIRECTIONAL.md)

- **레짐 매트릭스** (VIX + KOSPI/S&P 50/200일선): 강세 넷 +50~60%(캡 +70%) / 중립 +30~45% / 경계 +10~25% / 위기 −10~+10%
- **베타 관리**: 목표 0.5~0.8(정상 국면). 고β 리더 롱(1.1~1.4) + 저β 캐리 밸러스트 + 구조적 열위 숏(0.9~1.15), 미세조정은 **지수선물(G4 오버레이)로만** — 알파 보존 원칙
- **조건부 리밸런싱**: 주간 정기 + 즉시 트리거(200DMA 돌파/이탈, VIX +5pt·30·40, 지수 ±3%). 재위험 인수는 주 +15%p 램프
- **손실 사다리**: 일간 −2% → 넷 50% / 월중 −4% → 잔여월 넷 50% / 월중 −6% → 넷 0(뉴트럴 전환) / 고점 −10% → 청산 심사
- VaR 한도 1.8% = 알파 0.8% + 방향성 1.0% 분리 계측
- **현재 포지셔닝**(7/10 실측 신호로 판정): 중립 레짐 — 넷 +45%, β 0.46, Gross 71%

## 4. 대시보드 기능 — [index.html](index.html)

- **전략 A/B 탭 전환** — 아이디어·KPI·포지션·팩터·백테스트까지 북 단위 전면 전환
- **오늘의 투자 아이디어** — 최상단, 매일 갱신 (근거 필수, 태그: 시장뷰/신규진입/청산/리스크/레짐판정)
- **KPI**: NAV, 일간 P&L, ITD, Gross, Net(β조정), VaR 사용률, **변동성(연환산 — 실현치 우선, 부족 시 포지션 기반 추정)**
- **종목 상세 모달** (종목명 클릭): 시총(현지통화 + 원화 환산), **PER(12M forward, 야후 컨센서스)**, PSR·**매출총이익률(GPM)·영업이익률(OPM)**(T12M), 연간 3개년(매출/총이익/영업이익/순이익 + 마진), 분기 5개(전년 동기 YoY 비교), 우선주는 보통주 대비 디스카운트 표시
- **가상 매매 What-if 시뮬레이터**: 종목·±비중 입력 → Gross/Net/β/VaR/변동성 before→after 비교, 한도 초과 경고. 화면 전용(실반영은 data.js로만). VaR 추정 모델: √[(Gross×0.88bp)² + (|β|×2.33×σ_idx1.0%)²]
- **사후검증 차트**: NAV 3계열(A) / B vs A + 넷 레짐 밴드(B). **축은 데이터 기준**, 범위 밖 한도선은 가장자리 화살표
- **다크/라이트 모드**: 허브와 `pf_theme` localStorage 키 공유, 무플래시 초기화, 차트 SVG 색도 CSS 변수로 테마 연동
- **캐시버스팅**: data.js/backtest_data.js/fundamentals.js를 `?v=분단위`로 로드 — 일일 갱신 즉시 반영

## 6. 일일 운용 사이클 — [DAILY_UPDATE.md](DAILY_UPDATE.md)

스케줄 작업 `longshortport-daily-update` (매일 ~07:30, 미국 마감 반영 후):

앱이 꺼져 있으면 다음 실행 시 밀린 업데이트가 돌아간다. 원칙: 스톱은 재량 없이 기계 집행, 아이디어는 근거 필수, NAV 이력 소급 수정 금지.

## 7. 주요 기술 결정 기록

1. **배포는 `longshort/` 하위 폴더** — 저장소 루트에 기존 대시보드 90여 파일이 있어 충돌 방지. 모든 푸시는 이 폴더만.
2. **시세/재무는 야후 비인증 API** — chart v8(시세), fundamentals-timeseries(재무). v7 quote·quoteSummary는 401 → **crumb 쿠키 플로우**로 forward PER만 취득. 시총 = 주가×기본주식수 근사(TSM은 ADR 수 기준, 삼전우는 주식수 하드코딩).
3. **백테스트는 순수 표준 라이브러리** — 의존성 없이 `python backtest.py` 한 번으로 차트 데이터까지 재생성.
4. **P&L 규약**: 현지통화 수익률(KRW 선물환 헤지 가정), 비용 = 거래 그로스 10bps + 숏 대차 연 1%.
5. **정직성 원칙**: 사후 편향·빈 북 페널티·미반영 요소(공매도 금지, 리콜, 롤비용)를 문서에 명시. 백테스트는 수익률이 아닌 룰 검증임을 반복 고지.
